# Оценка системы PIKURR

*Дата оценки: апрель 2026*

---

## Общая оценка

| Критерий | Оценка | Комментарий |
|---|---|---|
| Сложность | 7 / 10 | Два изолированных контура, ML-инференс, GEE, GeoServer ImageMosaic, WPS |
| Безопасность | 5 / 10 | Нет TLS, дефолтные пароли на проде, GeoServer открыт напрямую |
| Устойчивость | 7 / 10 | Watchdog + systemd, skip-логика в ETL, ON CONFLICT в SQL; нет retry в deliver.py |
| Полезность | 9 / 10 | Полный цикл от спутника до веб-карты с историей и фильтрацией |

### Что уже хорошо

- Чёткое разделение контуров — между ETL и фронтендом только ZIP-файл
- Skip-логика во всех ETL-задачах (идемпотентность, можно перезапускать)
- Watchdog как systemd-сервис с автовосстановлением
- SSH-ключи для rsync вместо паролей
- Credentials убраны из git
- Каскадные фильтры и история по годам на фронте
- Дашборд администратора с пошаговым статусом

## TODO: план совершенствования

### 🔴 Безопасность (критично для продакшна)

**1. HTTPS на nginx**
- Настроить Let's Encrypt через certbot для домена geobotany.xyz
- Добавить в `nginx.conf` редирект HTTP → HTTPS и SSL-секцию
- Обновить `WMS_BASE_URL` во фронтенде на `https://`

**2. Сменить все дефолтные пароли**
- PostGIS: `pikurr/pikurr` → случайный пароль (в `deliver.env` и `docker-compose.server.yml`)
- GeoServer: `admin/geoserver` → сложный пароль (в `docker-compose.server.yml` и `geoserver_data/security/`)
- Задокументировать в `deliver.env.example` что пароли обязательно меняются

**3. Закрыть прямой доступ к GeoServer**
- Порт 8090 сейчас торчит наружу — прямой доступ к панели администратора
- Убрать `ports: "8090:8080"` из `docker-compose.server.yml`, оставить только внутренний
- Весь трафик GeoServer — только через nginx (`/geoserver/`)

**4. Ограничить WFS/WPS только для внутреннего использования**
- WFS и WPS дают полный доступ к данным PostGIS без аутентификации
- Настроить в GeoServer Security → Services: WFS/WPS только для аутентифицированных

### 🟡 Устойчивость

**5. Retry в deliver.py при сбое GeoServer**
- Сейчас если GeoServer недоступен в момент доставки — пакет падает без повтора
- Добавить retry с exponential backoff для REST-вызовов к GeoServer

**6. Алерт если пакет завис в inbox**
- Watchdog видит новый ZIP, но если deliver.py упал — ZIP остаётся в inbox
- Добавить в watchdog Telegram-уведомление если пакет не обработан за N минут

**7. Healthcheck для deliver.py**
- Сейчас watchdog не отличает "deliver.py упал" от "обрабатывается долго"
- Добавить таймаут на `subprocess.run` и retry-счётчик

**8. Graceful shutdown watchdog**
- При `systemctl stop` watchdog убивается сразу, даже если deliver.py работает
- Добавить обработку `SIGTERM` — дождаться завершения текущей доставки

**9. Мониторинг дискового пространства**
- Tiles-кэш и dist/ могут вырасти до десятков гигабайт
- Добавить в дашборд или watchdog проверку свободного места с алертом
